.. _registries:

Registries and Marketplaces
===========================

The Zscaler MCP Server is published to **five** public registries. Each registry has its own discovery surface and its own install flow — pick the one that matches the MCP client you're using.

.. list-table::
   :header-rows: 1
   :widths: 25 30 45

   * - Registry
     - Best for
     - Install command
   * - `Cursor Marketplace <https://cursor.com/marketplace/zscaler>`_
     - Cursor IDE users
     - One-click "Add to Cursor" button
   * - `Claude Marketplace <https://claude.com/plugins/zscaler>`_
     - Claude Code CLI users
     - ``claude plugin install zscaler``
   * - `Official MCP Registry <https://registry.modelcontextprotocol.io/?q=zscaler>`_
     - Any spec-compliant MCP client
     - Discoverable via the official MCP registry
   * - `Docker MCP Hub <https://hub.docker.com/mcp/server/zscaler-mcp-server/overview>`_
     - Containerized deployments (any client)
     - ``docker pull zscaler/zscaler-mcp-server``
   * - `GitHub MCP Registry <https://github.com/mcp?search=zscaler>`_
     - GitHub Copilot users, MCP-compatible clients reading from GitHub
     - Surfaces via the ``server.json`` manifest at the repo root

.. _cursor-marketplace:

Cursor Marketplace
------------------

Direct link: `cursor.com/marketplace/zscaler <https://cursor.com/marketplace/zscaler>`_

The Cursor Marketplace listing publishes a one-click installer that registers the Zscaler MCP Server with your Cursor IDE installation. The marketplace handles:

- Adding the server to your Cursor MCP configuration
- Pulling the latest released version
- Surfacing the bundled Cursor plugin, which adds Zscaler-specific tool guidance and skill prompts

For manual installation (without the marketplace), see :ref:`cursor-plugin`.

.. _claude-marketplace:

Claude Marketplace
------------------

Direct link: `claude.com/plugins/zscaler <https://claude.com/plugins/zscaler>`_

The Claude plugin marketplace publishes the Zscaler plugin for Claude Code (the CLI). Install with:

.. code-block:: bash

   claude plugin install zscaler

The plugin auto-registers the MCP server and installs the bundled Claude Code skills. For configuration details, see :ref:`claude-code-plugin`.

.. _claude-desktop-extension-registry:

Claude Desktop Extension (separate from the marketplace)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

The Claude **Desktop** application uses a different distribution mechanism — a single ``.mcpb`` bundle file that you drag into Claude Desktop. See :doc:`../guides/claude-desktop-extension` for the install walkthrough.

.. _official-mcp-registry:

Official MCP Registry
---------------------

Direct link: `registry.modelcontextprotocol.io <https://registry.modelcontextprotocol.io/?q=zscaler>`_

The Model Context Protocol consortium maintains the canonical registry of MCP servers. The Zscaler server is published with both a PyPI package descriptor and a Docker image descriptor:

- **PyPI:** ``uvx zscaler-mcp`` is the entry point
- **Docker:** ``zscaler/zscaler-mcp-server:latest`` is the image

Publication is driven by ``server.json`` at the repo root and is automated via ``mcp-publisher`` on every release. The package ownership proof lives in:

- ``README.md`` (``<!-- mcp-name: io.github.zscaler/zscaler-mcp-server -->``)
- ``Dockerfile`` (``LABEL io.modelcontextprotocol.server.name="io.github.zscaler/zscaler-mcp-server"``)

For the publication workflow, see ``integrations/github/README.md`` in the repository.

.. _docker-mcp-hub:

Docker MCP Hub
--------------

Direct link: `hub.docker.com/mcp/server/zscaler-mcp-server <https://hub.docker.com/mcp/server/zscaler-mcp-server/overview>`_

Docker maintains a curated MCP server catalog. The Zscaler image is published with:

- ``latest`` tag — most recent release
- ``X.Y.Z`` tags — each release version (e.g. ``0.12.2``)
- ``master`` tag — bleeding edge from the ``master`` branch

The image works with every MCP client that supports the ``docker`` transport pattern:

.. code-block:: json

   {
     "mcpServers": {
       "zscaler-mcp-server": {
         "command": "docker",
         "args": [
           "run", "-i", "--rm",
           "--env-file", "/absolute/path/to/.env",
           "zscaler/zscaler-mcp-server:latest"
         ]
       }
     }
   }

For HTTP-transport deployment (Bedrock AgentCore, Cloud Run, Container Apps, Kubernetes), see the corresponding deployment guide.

.. _registry-github:

GitHub MCP Registry
-------------------

Direct link: `github.com/mcp?search=zscaler <https://github.com/mcp?search=zscaler>`_

GitHub maintains an MCP-server index that reads the ``server.json`` manifest from a repository's root. The Zscaler entry surfaces in:

- GitHub Copilot's MCP server picker
- Any client that reads from the GitHub registry surface

The manifest declares both a PyPI runtime (``uvx``) and a Docker runtime, so clients can pick whichever matches their environment.

Required environment variables (stdio transport): ``ZSCALER_CLIENT_ID``, ``ZSCALER_CLIENT_SECRET``, ``ZSCALER_CUSTOMER_ID``, ``ZSCALER_VANITY_DOMAIN``.

Picking the right registry
--------------------------

.. list-table::
   :header-rows: 1
   :widths: 25 25 50

   * - You're using …
     - Pick …
     - Why
   * - Cursor IDE
     - Cursor Marketplace
     - Bundled Cursor plugin, one-click install
   * - Claude Code CLI
     - Claude Marketplace
     - ``claude plugin install zscaler``, bundled skills
   * - Claude Desktop
     - :doc:`../guides/claude-desktop-extension`
     - ``.mcpb`` bundle, no separate plugin install needed
   * - GitHub Copilot
     - GitHub MCP Registry
     - Native integration with the Copilot MCP picker
   * - VS Code / Gemini CLI / Kiro / other generic MCP clients
     - Docker MCP Hub or PyPI directly
     - Universal — works with any MCP-compliant client
   * - A managed cloud deployment (Cloud Run, Bedrock, Container Apps)
     - Docker MCP Hub
     - Pull the image, point your cloud runtime at it
